# 読書ログアプリ

## ①課題名
読書ログアプリ

## ②課題内容（どんな作品か）
- 本のタイトル、著者、読書状況、ひとことメモを入力して保存できるアプリです。
- 保存した読書ログは一覧で見られます。
- さくらサーバー上のMySQLにデータを保存し、画面で読み返せるようにします。
- PHPとPDOでデータベース接続をしています。

## ③アプリのデプロイURL
- https://limejackal58.sakura.ne.jp/reading_log/index.php

## ④アプリのログイン用IDまたはPassword（ある場合）

- PW: なし

## ⑤工夫した点・こだわった点
- 入力フォームと一覧のデザインをわかりやすく、読みやすくしました。
- `select.php` で保存したログがカード形式で表示されるようにしました。
- `functions.php` で共通のDB接続をまとめ、コードを再利用しやすくしました。
- サクラサーバー上で動くように、データベース接続情報とサーバ環境を調整しました。
- PHPのエラー表示を有効にして、サーバ側の問題を見つけやすくしています。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- さくらサーバーのPHP環境とローカル環境の違いを理解するのが難しかったです。
- デプロイ先のパスとファイルの配置場所を正しく合わせる必要がありました。
- 次回はログイン機能を追加して、ユーザーごとに読書ログを管理できるようにしたいです。
- 編集・削除機能を追加して、過去の記録を更新できるようにしたいです。
- カテゴリや評価などで検索できる機能も追加したいです。
