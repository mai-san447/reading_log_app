# ログイン機能の作り方（中学生でもわかる手順書）

このファイルは「Book Bank（ブックログ）にログイン機能をどうやって付けたか」を、あとから自分でも分かるようにまとめたものです。
PHP04で習った **session（ログイン状態の記録）** と **hash（パスワードの暗号化）** を、自分のアプリに応用しています。

---

## 0. まず「ログイン」ってどういう仕組み？

たとえるなら、会員制のお店です。

| 部品 | たとえ | ファイル |
|---|---|---|
| 会員登録 | 会員証をつくる受付 | `register.php` / `register_act.php` |
| ログイン | 会員証を見せて中に入る入口 | `login.php` / `login_act.php` |
| ログアウト | 出口（会員証を返す） | `logout.php` |
| 門番 | 各部屋の前で会員証を確認する人 | `functions.php` の `loginCheck()` |

ルールは2つだけ：
1. **パスワードはそのまま保存しない**。暗号化（ハッシュ化）してから保存する。
2. **各ページの先頭に「門番」を置く**。ログインしていない人は入口へ追い返す。

ファイルの名前の決まり：
- `〇〇.php` … **見た目**（フォームなどの画面）
- `〇〇_act.php` … **処理**（保存する・照合する。act = action＝動く係）

## 1. ユーザーの保管庫（テーブル）を用意する

会員証を保存する「表（テーブル）」= `gs_user_table` が必要です。
このアプリは `functions.php` の `connectDb()` が、テーブルを**自動で作る**しくみなので、そこに1つ足すだけです。

`functions.php` の `connectDb()` の中（`gs_reading_log` を作っている場所の下）に追加：

```php
// ★ログイン用：ユーザーを保管する表を自動で作る
$pdo->exec(
    "CREATE TABLE IF NOT EXISTS gs_user_table (
        id INT(11) NOT NULL AUTO_INCREMENT PRIMARY KEY,
        login_id VARCHAR(255) NOT NULL UNIQUE,
        login_pw VARCHAR(255) NOT NULL,
        created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8;"
);
```

- `CREATE TABLE IF NOT EXISTS` = 「もし無ければ作る」。すでにあれば何もしないので安全。
- `login_id` に `UNIQUE` = 同じIDを2人が使えないようにする鍵。
- `login_pw` にはパスワードの**暗号文**が入る（次のステップで暗号化する）。

## 2. 門番 `loginCheck()` をつくる

`functions.php` の一番下に、門番の関数を追加します。

```php
// ★門番：ログインしているか確認する。していなければログインページへ追い返す
function loginCheck()
{
    // セッションがまだ始まっていなければ始める
    if (session_status() === PHP_SESSION_NONE) {
        session_start();
    }

    // 合言葉（chk_ssid）が、いまのセッションIDと一致しなければ「入っちゃダメ」
    if (!isset($_SESSION['chk_ssid']) || $_SESSION['chk_ssid'] !== session_id()) {
        header('Location: login.php');
        exit();
    }
}
```

- `session_start()` = ログイン状態のメモ帳を開く（使う前に必ず必要）。
- `$_SESSION['chk_ssid']` = ログイン成功時に入れておく「合言葉」。
- 合言葉が今のセッションIDと合わなければ `login.php` へ `header('Location: ...')` で送り返す。
- `exit();` を忘れると、追い返したあとも続きが動いてしまうので必ず書く。

## 3. 会員登録ページ（`register.php` と `register_act.php`）

### 3-1. 画面 `register.php`（フォーム）

ポイントは1行だけ：`<form action="register_act.php" method="post">`。
「入力したIDとパスワードを、`register_act.php` に post（送る）」という意味です。

```html
<form action="register_act.php" method="post">
    <label for="login_id">ユーザーID</label>
    <input type="text" id="login_id" name="login_id" required>

    <label for="login_pw">パスワード</label>
    <input type="password" id="login_pw" name="login_pw" required>

    <button type="submit">登録する</button>
</form>
```

### 3-2. 処理 `register_act.php`（保存する係）

流れは「空チェック → 重複チェック → 暗号化 → 保存」。

```php
<?php
require_once __DIR__ . '/functions.php';

// フォーム以外（URL直打ち）で来たら追い返す
if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    header('Location: register.php');
    exit();
}

$login_id = trim((string)($_POST['login_id'] ?? ''));
$login_pw = (string)($_POST['login_pw'] ?? '');

// 1. 空っぽチェック
if ($login_id === '' || $login_pw === '') {
    header('Location: register.php?error=empty');
    exit();
}

$pdo = connectDb();

// 2. 同じIDが無いか確認
$stmt = $pdo->prepare('SELECT id FROM gs_user_table WHERE login_id = :login_id');
$stmt->bindValue(':login_id', $login_id, PDO::PARAM_STR);
$stmt->execute();
if ($stmt->fetch()) {
    header('Location: register.php?error=exists');
    exit();
}

// 3. パスワードを暗号化（★hash.phpで習ったやつ！）
$hash = password_hash($login_pw, PASSWORD_DEFAULT);

// 4. 保存
$stmt = $pdo->prepare('INSERT INTO gs_user_table (login_id, login_pw) VALUES (:login_id, :login_pw)');
$stmt->bindValue(':login_id', $login_id, PDO::PARAM_STR);
$stmt->bindValue(':login_pw', $hash, PDO::PARAM_STR);
$stmt->execute();

// 5. 登録できたらログインページへ
header('Location: login.php?registered=1');
exit();
```

- `password_hash()` = パスワードをぐちゃぐちゃの暗号文に変える（`hash.php` と同じ）。
- `prepare` と `bindValue` = 悪い人がSQLを壊せないようにする安全な書き方（SQLインジェクション対策）。
- `?error=empty` のような印をURLに付けて、`register.php` 側でメッセージを出し分けています。

## 5. ログイン処理 `login_act.php`（照合する係）

いちばん大事な部分です。「IDで探す → パスワードを照合 → 合言葉を保存」。

```php
<?php
require_once __DIR__ . '/functions.php';
session_start();

if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    header('Location: login.php');
    exit();
}

$login_id = trim((string)($_POST['login_id'] ?? ''));
$login_pw = (string)($_POST['login_pw'] ?? '');

$pdo = connectDb();

// 1. 入力されたIDのユーザーを探す
$stmt = $pdo->prepare('SELECT * FROM gs_user_table WHERE login_id = :login_id');
$stmt->bindValue(':login_id', $login_id, PDO::PARAM_STR);
$stmt->execute();
$user = $stmt->fetch(PDO::FETCH_ASSOC);

// 2. ユーザーがいて、パスワードも合っていれば成功
if ($user && password_verify($login_pw, $user['login_pw'])) {
    session_regenerate_id(true);          // なりすまし対策：IDを作り直す
    $_SESSION['chk_ssid'] = session_id(); // 合言葉を保存（門番が確認する）
    header('Location: select.php');       // 本棚へ
    exit();
}

// 3. 失敗したらログイン画面へ
header('Location: login.php?error=1');
exit();
```

- **ここがPHP04との違い（重要）**：DBには暗号文しか入っていないので、`WHERE login_pw = ...` で直接比べるのではなく、
  IDだけで探して、`password_verify(入力したパスワード, 保存した暗号文)` で照合します。合っていれば `true`。
- `session_regenerate_id(true)` = ログインの瞬間にセッションIDを新しくして、なりすまし（セッション固定攻撃）を防ぐ。

## 6. ログアウト `logout.php`

会員証を返す（ログイン状態を消す）だけ。

```php
<?php
require_once __DIR__ . '/functions.php';
session_start();

$_SESSION = [];                 // 中身を空に
if (isset($_COOKIE[session_name()])) {
    setcookie(session_name(), '', time() - 42000, '/'); // クッキーも消す
}
session_destroy();              // セッションを破棄
header('Location: login.php');  // ログイン画面へ
exit();
```

## 7. 各ページに門番を立てる（いちばん大事）

守りたいページの**先頭**で `loginCheck()` を呼ぶだけ。これを忘れると、ログインしなくても中身が見えてしまいます。

`select.php` など、`<?php` で始まるページ：

```php
<?php
require_once __DIR__ . '/functions.php';
loginCheck(); // ← これを足すだけ！
$pdo = connectDb();
```

`index.php` のように `<!DOCTYPE html>`（HTML）で始まるページは、**その前に**PHPのかたまりを置きます：

```php
<?php
require_once __DIR__ . '/functions.php';
loginCheck();
?>
<!DOCTYPE html>
...
```

> ⚠️ 注意：`loginCheck()` は `header()` でページを移動させます。`header()` は**画面に文字が1文字でも出る前**に呼ぶ必要があるので、必ず**ファイルの一番上**に置くこと。

門番を立てたページ：`index` / `select` / `analytics` / `log` / `edit` / `insert` / `update` / `delete`。
（`login` / `register` / `logout` には**立てない**。立てると入口にすら入れなくなるため。）

最後に、メインページのヘッダーに出口も付けました：
```html
<a href="logout.php">ログアウト</a>
```

## よくあるエラーと対処

| 症状 | 原因 | なおし方 |
|---|---|---|
| `headers already sent` | `loginCheck()` の前に文字が出力されている | `loginCheck()` をファイルの一番上（`<?php` の直後）に置く |
| ずっとログイン画面に戻る | 合言葉が保存されていない／`session_start()`忘れ | `login_act.php` で `session_start()` があるか確認 |
| DB接続エラー | MySQLが起動していない | XAMPPで MySQL を Start |
| ログインできない | パスワードを平文で保存してしまった | `password_hash` で登録し直す（`password_verify` は暗号文とだけ照合できる） |

## 用語ミニ辞典

- **session（セッション）**：ページを移動しても消えない、その人専用のメモ帳。ログイン状態の記録に使う。
- **hash（ハッシュ）**：パスワードを元に戻せない暗号文に変えること。漏れても安全。
- **`password_hash` / `password_verify`**：暗号化する係 / 照合する係。ペアで使う。
- **`header('Location: ...')`**：別のページへ強制的に移動させる命令。
- **prepare / bindValue**：SQLを安全に組み立てる書き方（悪用対策）。
